# Roadmap

## Product Principle

Models propose. Tools decide. The platform must produce either a reproducible verified Git diff or an honest evidence-backed failure report.

## Stages

1. **Deterministic Orchestration Foundation** - Completed bootstrap: Python project, CLI, typed config, run artifacts, named command execution, Git worktree foundation, validation workflow.
2. **RTL Repository Discovery and Structured Repository Model** - Completed deterministic repository scanner, lightweight SystemVerilog extraction, hierarchy candidate scoring, build-flow evidence discovery, JSON repository map, CLI, run artifacts, and tests.
3. **Issue Parsing and Task Contracts** - Completed deterministic issue-to-contract parser with explicit scope, invariants, acceptance criteria, validation commands, prohibited shortcuts, evidence requirements, warnings, CLI, and tests.
4. **Model Provider Abstraction and One Bounded Implementation Agent** - Completed typed provider interface, deterministic stub provider, bounded implementation loop, explicit permissions, structured tool calls, named-command validation, run artifacts, reports, CLI, and tests.
5. **Verification Execution and Failure Iteration** - Completed deterministic validation classification, bounded retry iteration, concise failure evidence, retry/stop decisions, report fields, CLI summary, examples, and tests.
6. **Independent Reviewer** - Completed read-only review reports with deterministic findings, optional provider finding separation, evidence citations, validation gating, scope checks, CLI, examples, and tests.
7. **Waveform and Assertion Triage** - Completed bounded assertion, simulator-context, waveform-reference, and missing-waveform triage from command artifacts with review integration, CLI, examples, and tests.
8. **Verification Strength and Mutation Assessment** - Completed deterministic artifact-only verification-strength scoring, weak validation pattern detection, CLI, README usage, and tests. Mutation execution remains intentionally out of scope.
9. **Benchmark Suite** - Completed deterministic local benchmark manifests, bounded named-command runner, expected-outcome reporting, run artifacts, README usage, and tests.
10. **Evidence Bundle Export** - Completed deterministic local evidence-bundle indexes with artifact provenance, hashes, schema versions, omitted-artifact references, CLI usage, and tests.
11. **Schema Examples and Compatibility Fixtures** - Completed compact checked-in public-schema examples plus compatibility tests that validate them through current typed models.
12. **CLI Documentation Consistency Pass** - Completed README cleanup, installed-CLI command verification, source-tree invocation guidance, and README command help coverage tests.
13. **Packaging Smoke Verification** - Completed bounded local wheel/install smoke verification for console-script and module invocation as part of the canonical check workflow.
14. **Compact End-to-End Example Check** - Completed compact local scripted example check across checked-in fixtures, deterministic implementation retry, review, verification-strength, benchmark, evidence export, and schema-backed artifact assertions.
15. **Failure Report Example Check** - Completed compact local scripted example check for the honest terminal-failure path, failed report artifact, command evidence, review disposition, verification-strength result, and evidence-bundle export.
16. **Tool Failure Report Example Check** - Completed compact local scripted example check for deterministic structured-tool failure reporting, failed tool-result evidence, absent validation execution, review disposition, verification-strength result, and evidence-bundle export.
17. **Example Script Helper Consolidation** - Completed shared `scripts/_example_check.py` helper for repository root, venv-aware interpreter, source-path setup, and the `run_cli` CLI subprocess helper; the end-to-end, failure, and tool-failure example scripts now reuse it without changing workflow behavior.
18. **No-Change Implementation Example Check** - Completed compact local example check for the deterministic no-op (no-change) implementation path, covering a successful `replace_text` application with identical old/new content that still ends in an unacceptable review and insufficient verification strength because no validation command ran.
19. **VCD Failure Window Extraction** - Completed the first deterministic RTL failure-intelligence capability: typed, versioned waveform-slice schema, deterministic textual-VCD parser and bounded window extractor, `extract-waveform-window` CLI, source metadata with SHA-256 and parse statistics, accurate scalar/vector/`x`/`z` representation, pre-window initial values, optional triage-report source resolution, checked-in VCD fixtures, README example, and tests. No causal or root-cause interpretation; textual VCD only.
20. **Assertion-to-Waveform Failure Linking** - Completed the deterministic linkage from triaged assertion failures to bounded VCD slices: typed, versioned linkage report, `link-assertion-waveform` CLI, stable assertion selection by id/index, timescale-aware simulator-time-to-tick conversion, reuse of the existing window extractor (via a shared `read_vcd_timescale` helper), honest failures for missing/ambiguous timestamps, multiple/missing/unsupported/malformed waveforms, a runnable triage fixture, README example, and tests. Never infers root cause; never silently resolves ambiguity.
21. **Waveform Evidence Bundle Integration** - Completed: the deterministic evidence-bundle exporter now recognizes, classifies, hashes, and records schema versions for waveform-slice and assertion-link artifacts under a run directory, reusing existing provenance and omitted-content handling with no export redesign or artifact-schema change.
22. **Automatic Relevant-Signal Reduction** - Completed: deterministic, bounded, evidence-ranked reduction of a waveform slice to the signals most relevant to a failure, via a typed versioned report, a `reduce-signals` CLI, and a reduced waveform-slice artifact. Ranks only from explicit textual/transition evidence (assertion-named, transition at/near failure, `x`/`z` presence, parent-scope hierarchy proximity); strict subset, per-criterion reasons, honest empty/warning handling. No dependency tracing, semantic interpretation, source localization, stimulus minimization, or causal claims.
23. **Passing-vs-Failing Waveform Comparison** - Completed: deterministic comparison of a failing waveform slice against a passing (reference) slice over shared signals and a common time basis, via a typed versioned report and a `compare-waveforms` CLI. Reports shared/added/removed signals, per-signal first divergence time and values, transition counts, x/z differences, divergence duration/intervals, global earliest divergence, and a deterministic divergence ranking; explicit time basis with recorded normalization and honest warnings for incompatible timescales/windows/ambiguous names. Reuses existing slice artifacts; no causal claims, dependency tracing, semantic interpretation, source localization, or stimulus minimization.
24. **Signal-to-RTL Source Mapping** - Completed: deterministic, evidence-based mapping of hierarchical waveform signal names to candidate RTL declarations (module/interface/package/program/checker with file and line) from the existing repository map, via a typed versioned report and a `map-signals` CLI. Distinguishes exact / probable / ambiguous / unresolved with tiered per-candidate scores and explicit reasons; preserves all candidates under ambiguity. No semantic elaboration, connectivity/driver tracing, model-based analysis, or causal claims.
25. **Static RTL Driver and Dependency Tracing** - Completed: bounded, deterministic, explicitly-textual driver/dependency extraction for mapped signals via a typed versioned report and a `trace-drivers` CLI. Scans declaring RTL for continuous/procedural assignments and port connections (file, line, kind, text, LHS/RHS identifiers, enclosing declaration, best-effort guard); bounded upstream dependency expansion (configurable depth/nodes) with edges labeled textual / inferred_textual. Preserves multiple drivers and unresolved identifiers; masks comments/strings and strips based literals. No elaboration, simulation, semantic dataflow, or causal claims.
26. **Failure Divergence Graph** - Completed: deterministic, purely compositional graph rooted at the comparison's diverging signals, via a typed versioned report and a `divergence-graph` CLI. Nodes compose divergence (time/values/score), mapping status + declaration location, and driver-resolution status; edges are the driver-trace dependency edges retaining their textual/inferred-textual label and source citation. Bounded BFS from roots (configurable depth/nodes) over the existing driver-trace edges; preserves multiple drivers, unresolved identifiers, and warns on cross-artifact mismatch. No new RTL scanning, elaboration, semantic dataflow, ranking-as-causation, or root-cause claims.
27. **Failure Intelligence Evidence Bundle Integration** - Completed: the deterministic evidence-bundle exporter now recognizes, classifies, hashes, and records schema versions for the relevant-signal-reduction, waveform-comparison, signal-source-map, driver-trace, and failure-divergence-graph artifacts under a run directory, via five content-based key-set classifiers reusing existing provenance and omitted-content handling with no export redesign or artifact-schema change.
28. **Compact Failure Intelligence Example Check** - Completed: a deterministic local integration check (`scripts/failure_intelligence_example_check.py`) chains the real CLI end-to-end (waveform extraction → relevant-signal reduction → passing/failing comparison → repository discovery → signal-source mapping → static driver tracing → failure-divergence graph → evidence-bundle export) over checked-in fixtures, adds a compact `examples/waveforms/passing.vcd` reference, asserts stable semantic properties via the existing schemas, and is registered in `scripts/check.py`.
29. **Compact Failure Report Synthesis** - Completed: a deterministic, purely compositional `synthesize-failure-report` command composes the failure-divergence-graph report (plus optional reduction, driver-trace, verification-strength, and review reports) into a typed versioned failure report and a concise engineer-facing Markdown summary, separating observed facts, earliest divergence, ranked signals, candidate source locations, textual driver/dependency evidence, unresolved/ambiguous evidence, verification/review status, and artifact provenance; every statement cites its originating artifact and nothing is labeled a root cause.
30. **Failure Intelligence Run Orchestration** - Completed: a bounded, deterministic `run-failure-intelligence` command invokes the existing stages in a fixed sequence (waveform extraction → comparison → discovery → mapping → driver tracing → divergence graph → reduction → failure-report synthesis), reuses each stage's service with no duplicated logic, persists every artifact under one `RunStore` directory, and emits a typed versioned run manifest recording per-stage status/inputs/outputs/durations/warnings/failure reason and linking all artifacts; it stops honestly on terminal errors while preserving completed artifacts.
31. **Failure Intelligence Run Resume and Replay** - Completed: the `run-failure-intelligence` command gained deterministic, manifest-driven `--resume` and `--replay-from <stage>`, validating reused artifacts (existence, recorded SHA-256, typed model, supported schema version, matching run inputs) before trusting them and regenerating otherwise, with downstream cascade invalidation. The run-manifest schema was bumped to v2 (per-stage disposition executed/reused/regenerated/skipped/failed; per-artifact SHA-256), skipped stages are recorded after a failure, and events explain every reuse/invalidation decision. One execution path, no new analysis behavior.
32. **Failure Intelligence Run Portability and Relative Provenance** - Completed: run directories are portable — stage inputs/outputs are recorded as typed `PathRef`s marked `run_relative` or `external`, external run inputs are recorded explicitly with existence, run-relative artifacts resolve against the current run directory (so moved/copied runs resume and replay), a `resolve_run_relative` helper rejects traversal/escaping paths, unsafe recorded paths force regeneration, missing external inputs fail honestly, and the manifest schema was bumped to v3 while still accepting v2/v3 for reuse (no migration framework).
33. **Failure Intelligence Run Inspection and Validation** - Completed: a read-only `inspect-run` command validates a run directory against its manifest without re-running anything — resolving run-relative artifacts against the actual directory, checking existence, recorded SHA-256, typed model, and schema version, and reporting per-artifact validity (valid/missing/hash_mismatch/schema_malformed/schema_unsupported/unsafe_path), per-stage validity (valid/incomplete/stale/invalid), external-input existence, and overall run validity via a typed versioned report; exits non-zero on an invalid run and 2 on an unreadable manifest.
34. **Portable Failure Package Export** - Completed: a read-only, inspection-gated `export-failure-package` command packages a validated run directory into a single self-contained portable directory package (run manifest, inspection report, JSON/Markdown failure report, and validated manifest-referenced artifacts under `run/`), indexed by a typed versioned package manifest recording each file's package-relative path, role, size, SHA-256, schema version, and run-relative provenance; refuses invalid runs (only `--allow-failed` exports a failed-but-consistent run, marked `failed`), excludes external inputs/logs/caches/unrelated files, keeps paths traversal-safe, and verifies the package before success.
35. **AXI Router Seeded-Failure Validation** - Completed: added a compact checked-in AXI-stream-router fixture (`examples/axi-stream-router/`) whose RTL drives real internal signals, plus a seeded failing-vs-passing VCD pair (payload instability under backpressure) and a scripted check (`scripts/axi_router_seeded_failure_check.py`) that drives the real orchestrator, `inspect-run`, and `export-failure-package` and asserts the pipeline localizes the seeded divergence to genuine driver evidence (earliest signal/time, ranked signals, mapped module/file, cited `assign` driver and dependency chain, connected graph, failure report, portable package) while preserving ambiguity and avoiding causal claims.
36. **Real AXI Router Repository Pilot** - Completed: added a compact multi-file repository (`examples/axi-router-repo/`) — a top module `axi_router` instantiating two child modules (`axi_ingress`, `axi_route`) across separate files — with a seeded cross-module failing-vs-passing VCD pair and a scripted check (`scripts/axi_router_repository_pilot_check.py`) that drives the real orchestrator, `inspect-run`, and `export-failure-package` and asserts the existing pipeline resolves signals to the correct child files, reconstructs the `payload_out → payload_staged → payload_in` driver chain with links cited to two different files, connects the divergence graph across files, localizes the earliest divergence, and exports a portable package — all with no new analysis behavior.
37. **Cross-Module Ambiguity and Multi-Instance Robustness Pilot** - Completed: added a deliberately ambiguous fixture (`examples/axi-router-ambiguity/`) — a child module `lane` defined in two files and instantiated more than once, with non-unique internal signal names — and a scripted check (`scripts/axi_router_ambiguity_pilot_check.py`) that drives the real pipeline and asserts source mapping, driver tracing, the divergence graph, and the synthesized report all report the divergent signal as `ambiguous` and preserve both candidate files (not a false-confident single answer) while still localizing the seeded divergence — with no new analysis behavior.
38. **Simulator-Generated AXI Failure Pilot** - Completed: added a simulatable design + testbench (`examples/axi-router-sim/`) with a compile-time seeded fault and a gated check (`scripts/axi_router_simulated_failure_check.py`) that, when Icarus Verilog is present, compiles and runs the same stimulus clean and faulted to emit a genuine passing-vs-failing VCD pair, drives the existing pipeline over it, and asserts the divergence is localized to `axi_pipe.sv`; the check skips cleanly when the simulator is absent so the suite stays hermetic, and the simulator is never a product runtime dependency.
39. **Simulator-Generated Multi-Module Failure Pilot** - Completed: added a hierarchical simulatable design (`examples/axi-router-sim-hier/`) — a top module instantiating `ingress` and `route` child modules across separate files with a compile-time seeded fault that originates in the ingress child and propagates into the route child's output — and a gated check (`scripts/axi_router_simulated_multimodule_check.py`) that, when Icarus Verilog is present, generates the passing/failing VCD pair and drives the existing pipeline, asserting cross-file source mapping and the cross-module `payload_out → payload_staged → data_in` driver chain (cited across both files) localize the seeded divergence; skips cleanly when the simulator is absent.
40. **Simulator Failure Triage Integration Pilot** - Completed: added a simulatable fixture (`examples/axi-router-sim-triage/`) whose testbench emits a stable timestamped assertion marker and terminates non-zero, and a gated check (`scripts/axi_router_simulated_triage_check.py`) that runs the failing simulation through the existing command runner, triages the captured result, links the assertion to the generated waveform (deriving the failure time `45 ns`→tick 45 and the VCD path with no manual input), and drives the existing failure-intelligence orchestration to localize the divergence to `axi_pipe.sv` — asserting the triage artifact and the localized divergence are produced and consistent; skips cleanly when the simulator is absent.
41. **External AXI Router Repository Integration** - Completed vendored, pinned, license-compatible verilog-axis subset and a gated external-repository check that drives existing discovery, signal-source mapping, and driver-tracing services over real third-party RTL with provenance/drift validation and no network access during canonical checks.
42. **External RTL Mapping Accuracy Follow-up** - Active next milestone: fix the real-code accuracy gaps exposed by the external repository pilot: declaration line reporting on leading blank/comment regions, nested instance-path primary candidate selection, and cross-file dependency-node identity.
43. **Prohibited-Shortcut Review Finding Example Check** - Deferred: add a compact local example check exercising the existing but currently untested `det-prohibited-shortcut-N` review finding, using a deliberate diff that textually conflicts with a task-contract prohibited shortcut. Remains intentionally deferred.
